### 一、 DV (距离向量) 和 LS (链路状态) 的特点

**习题 1 (选择题):**
关于距离向量 (DV) 算法和链路状态 (LS) 算法的区别，下列说法**错误**的是：
A. DV 算法中，路由器只知道它的邻居，而不知道整个网络的拓扑结构。
B. LS 算法中，每个路由器都拥有整个网络的完整拓扑图。
C. DV 算法容易产生“计数到无穷大” (Count-to-infinity) 的问题，收敛较慢。
D. RIP 协议是典型的 LS 协议，而 OSPF 是典型的 DV 协议。

> **答案:** D
> **解析:**
> *   **DV (如 RIP):** 路由器只和**邻居**交换信息（自己的路由表），视野受限（“谣言传播”），收敛慢，好消息传得快，坏消息传得慢（计数到无穷大问题）。
> *   **LS (如 OSPF):** 路由器向**全网**广播链路状态信息（LSA），每个节点都有全网拓扑图（LSDB），使用 Dijkstra 算法计算最短路径，收敛快，无环路。
> *   D 选项说反了，RIP 是 DV，OSPF 是 LS。

**习题 2 (简答题):**
请简述为什么链路状态 (LS) 路由协议比距离向量 (DV) 路由协议更适合大型网络？

> **答案:**
> 1.  **收敛速度:** LS 协议在网络拓扑发生变化时，能更快地收敛，减少路由振荡。
> 2.  **避免环路:** LS 协议基于全网拓扑计算（Dijkstra 算法），天然避免了路由环路；而 DV 协议容易出现环路（虽有水平分割等机制，但不如 LS 彻底）。
> 3.  **流量占用:** DV 协议（如 RIP）通常周期性广播整个路由表，浪费带宽；LS 协议仅在拓扑变化时触发更新，平时只发送轻量级的 Hello 包，更节省带宽。

### 二、 OSPF 协议、BGP 协议的基本过程

**习题 1 (填空题):**
OSPF 路由器在建立邻接关系的过程中，会经历多个状态。当两台路由器完成了 LSDB（链路状态数据库）的同步，确立了完全的邻接关系时，它们处于 \_\_\_\_\_\_\_\_\_ 状态。

> **答案:** Full
> **解析:** OSPF 邻居建立的关键状态流转：Down -> Init -> 2-Way (建立邻居) -> ExStart -> Exchange -> Loading -> **Full** (完全邻接，LSDB 同步完成)。

**习题 2 (选择题):**
关于 BGP (边界网关协议) 的基本过程和特点，下列描述正确的是：
A. BGP 是一种基于链路状态的内部网关协议 (IGP)。
B. BGP 建立在 UDP 之上，端口号 520。
C. BGP 使用 TCP 连接 (端口 179) 交换路由信息，主要用于不同自治系统 (AS) 之间的路由选择。
D. BGP 路由器之间周期性地交换整个路由表。

> **答案:** C
> **解析:**
> *   A 错：BGP 是外部网关协议 (EGP)，且是路径向量 (Path Vector) 协议。
> *   B 错：RIP 用 UDP 520。
> *   C 对：BGP 为了可靠性，使用 TCP 连接（端口 179）。
> *   D 错：BGP 仅在启动时交换全量路由，之后只触发式更新（只发增量/变化的部分）。

### 三、 IPv6 的表示和其他特点

**习题 1 (应用题 - 地址压缩):**
请将下列 IPv6 地址使用“零压缩”规则（双冒号 ::）化简为最短形式：
`2001:0DB8:0000:0000:0000:0000:1428:57ab`

> **答案:** `2001:DB8::1428:57ab`
> **解析:**
> 1.  去掉每组开头的零：`2001:DB8:0:0:0:0:1428:57ab`
> 2.  连续的一组或多组 `0` 可以用 `::` 代替，但只能使用一次：`2001:DB8::1428:57ab`

**习题 2 (多选题):**
与 IPv4 相比，IPv6 具有以下哪些新特点？
A. 地址长度扩展到 128 位。
B. 首部长度是固定的 (40 字节)，取消了首部校验和字段。
C. 取消了广播 (Broadcast) 模式，代之以组播 (Multicast) 和任播 (Anycast)。
D. 必须使用 DHCPv6 才能获取 IP 地址。

> **答案:** A, B, C
> **解析:**
> *   A, B, C 均是 IPv6 的核心改进。取消校验和是为了加快路由器处理速度（由链路层或传输层负责校验）。
> *   D 错：IPv6 支持 SLAAC (无状态地址自动配置)，设备可以根据网关通告和自己的 MAC 地址自动生成 IP，不需要 DHCP 服务器也能上网。

### 四、 NAT (网络地址转换)

**习题 1 (分析题):**
家庭宽带路由器通常使用 NAT 技术让多台内网设备共用一个公网 IP 上网。请问这种允许多个内部 IP 映射到同一个公网 IP 的不同端口的技术被称为什么？
A. 静态 NAT (Static NAT)
B. 动态 NAT (Dynamic NAT)
C. NAPT (网络地址端口转换) / PAT
D. 路由重分发

> **答案:** C
> **解析:**
> *   静态/动态 NAT 通常是 IP 一对一的映射。
> *   **NAPT (Network Address Port Translation)** 利用传输层的**端口号**来区分内网不同的主机，实现了“多对一”的复用，这是最常见的 NAT 形式。

**习题 2 (简答题):**
NAT 技术虽然解决了 IPv4 地址枯竭的问题，但它破坏了网络的什么原则？会对哪类应用产生影响？

> **答案:**
> *   **破坏的原则:** 端到端 (End-to-End) 通信原则。因为 NAT 设备修改了 IP 包头，外网主机无法直接主动发起连接访问内网主机。
> *   **影响的应用:** P2P (对等网络) 应用（如 BitTorrent）、VoIP 通话、某些即时通讯软件。这些应用通常需要通过“NAT 穿透”技术（如 STUN/TURN）来解决连通性问题。

### 五、 SDN (软件定义网络) 的基本概念

**习题 1 (选择题):**
SDN 的核心思想是将网络的 \_\_\_\_\_\_\_\_\_ 与 \_\_\_\_\_\_\_\_\_ 分离。
A. 核心层；接入层
B. 控制平面 (Control Plane)；数据平面 (Data Plane)
C. 发送端；接收端
D. 硬件；软件

> **答案:** B
> **解析:** SDN 的最根本特征就是**转控分离**，即把负责路由计算和决策的“控制平面”从网络设备中剥离出来，集中到控制器上；底层的交换机只负责单纯的“数据平面”转发。

**习题 2 (判断题):**
在 SDN 架构中，南向接口 (Southbound Interface) 用于控制器与上层应用程序之间的通信，而北向接口 (Northbound Interface) 用于控制器与底层交换机之间的通信。

> **答案:** 错误
> **解析:** 说反了。
> *   **南向接口 (如 OpenFlow):** 控制器 <-> 底层设备（交换机）。(向下控制硬件)
> *   **北向接口 (如 REST API):** 控制器 <-> 上层应用。(向上提供服务)

### 六、 网络层的流量控制相关内容

*注：通常流量控制(Flow Control)属于传输层(TCP)或链路层的功能。但在网络层，相关概念通常指“拥塞控制(Congestion Control)”或早期的 ICMP 源抑制。*

**习题 1 (选择题):**
在网络层，当路由器因为缓冲区满而丢弃数据包时，它可以向源主机发送哪种 ICMP 报文来要求源主机降低发送速率？(注：该机制在现代网络中已不常用，但属于经典考点)
A. ICMP Redirect (重定向)
B. ICMP Time Exceeded (超时)
C. ICMP Source Quench (源点抑制)
D. ICMP Destination Unreachable (目标不可达)

> **答案:** C
> **解析:**
> *   **ICMP Source Quench (源点抑制):** 专门设计用于网络层的流量/拥塞控制。当路由器拥塞时，发送此报文给源主机，要求其减慢发送速度。

**习题 2 (简答题):**
简述网络层的“漏桶算法 (Leaky Bucket)”和“令牌桶算法 (Token Bucket)”在流量整形(Traffic Shaping)中的主要区别。

> **答案:**
> *   **漏桶算法:** 强行限制数据的**输出速率**是恒定的（平滑突发流量）。如果输入太快，桶满了就会丢包。它无法处理突发流量，只能平滑流量。
> *   **令牌桶算法:** 允许一定程度的**突发流量**。只要桶里有令牌，就可以发送数据。令牌按固定速率产生，如果一段时间没发数据，令牌积攒下来，之后就可以通过一波突发数据。
> *   **总结:** 漏桶控制**速率** (硬限制)，令牌桶控制**容量** (允许突发)。
