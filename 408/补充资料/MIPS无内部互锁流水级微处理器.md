**MIPS** 是 **Microprocessor without Interlocked Pipelined Stages**（无内部互锁流水级微处理器）的缩写。
[[mips指令集简写]]  
听起来很拗口？别担心，在计算机专业（特别是备考）的语境下，你只需要把它理解为：

**“计算机组成原理界的通用语”** 或者 **“教科书御用的标准处理器架构”**。

下面我分三个层面来帮你彻底理解它：

### 1. 它是什么？（本质）
MIPS 是一种 **指令集架构 (ISA, Instruction Set Architecture)**。

*   **比喻**：如果把 CPU 比作一个厨师，**ISA 就是“菜谱”**。
    *   Intel 的 x86 架构（你的电脑用的）是一本厚厚的、复杂的菜谱，里面有“红烧狮子头”、“佛跳墙”这种一步到位的复杂指令。
    *   **MIPS 架构** 是一本极简的菜谱，里面只有“切菜”、“倒油”、“炒肉”这种最基本的动作。
*   **RISC 理念**：MIPS 是 **精简指令集 (RISC)** 的代表。它的核心思想是：**指令越简单越好，每条指令长度一样（32位），执行速度越快越好。**

### 2. 为什么考试全是 MIPS？（重要性）
你可能会问：“现在的电脑都用 Intel 或者 ARM 芯片，为什么学校还教 MIPS？”

这是因为 **MIPS 设计得太完美、太整齐了，最适合用来教学。**

*   **Patterson & Hennessy 的“圣经”**：计算机体系结构的两位鼻祖（也是图灵奖得主）写了一本经典教材《计算机组成与设计》，全书就是用 MIPS 来讲解 CPU 是怎么造出来的。
*   **因为简单，所以好算**：
    *   你在题目里看到的“五级流水线”（取指、译码、执行、访存、写回），在 MIPS 上实现起来最标准、最清晰。
    *   如果是 Intel 的 x86 架构，一条指令长短不一，甚至有的指令要几百个周期才能跑完，根本没法拿来出手算的题目。

### 3. MIPS 的三个关键特征（考试常考的背景）

当你做这类题目时，你其实默认接受了 MIPS 的以下三个设定：

1.  **固定长度**：
    *   所有指令都是 **32位（4字节）** 长。
    *   *考点影响*：PC（程序计数器）每次加 4，而不是加 1，就是因为一条指令占 4 个字节。

2.  **Load/Store 架构（载入/存储架构）**：
    *   这是最核心的特征！**只有 `lw` (load) 和 `sw` (store) 能访问内存。**
    *   其他的运算指令（如 `add`, `sub`）只能在 **寄存器** 之间玩，不能直接去碰内存。
    *   *考点影响*：这就是为什么你的题目里 `lw` 指令最慢，因为它既要算地址，又要读内存。

3.  **通用寄存器堆**：
    *   MIPS 有 32 个通用寄存器（$0 - $31）。
    *   *考点影响*：题目中出现的 `rs`, `rt`, `rd` 实际上就是这 0-31 号寄存器的编号（占用指令中的 5 个 bit，因为 $2^5=32$）。

### 总结
在你的考试中，**MIPS 就是一个简化的、理想化的 CPU 模型**。

*   当你看到“MIPS”时，不要把它想成一个具体的芯片。
*   要把它想成一套**游戏规则**：规则规定了指令怎么写、流水线怎么流、寄存器怎么用。你刚才做的所有计算，都是在这个规则框架下进行的数学游戏。

**冷知识**：虽然现在电脑不用 MIPS，但以前的 **索尼 PS1、PS2 游戏机**，以及早期的 **路由器** 用的都是 MIPS 芯片。现在它的教学地位正在逐渐被 **RISC-V**（开源的新一代架构）取代，但目前的考试体系里，MIPS 依然是老大。
