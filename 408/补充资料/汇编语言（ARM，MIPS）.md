`$` 符号的意思非常简单：**它是一个“前缀”，用来告诉计算机“后面跟的是一个寄存器”**。

之所以第一个例子没有，第二个例子有，是因为这两个例子使用的是**两种不同的汇编语言“方言”（架构）**。

### 1. 为什么会有区别？

汇编语言（Assembly）并不是全世界统一的。不同的 CPU 架构（比如 Intel 的 x86、手机里的 ARM、教学常用的 MIPS）都有自己的语法规则。

*   **第一个例子 (`ADD R1, R2, R3`)：**
    *   这是**通用伪代码**（或者类似 **ARM** 风格的汇编）。
    *   在这种风格里，教科书为了让你看得懂，直接用 `R1`, `R2` (Register 1, Register 2) 来代表寄存器。
    *   **特点：** 简洁，适合讲原理，不需要特殊符号。

*   **第二个例子 (`lw $t0, 0($s0)`)：**
    *   这是标准的 **MIPS** 汇编语言。
    *   **MIPS 的语法规定：** 所有引用寄存器的地方，**必须**加 `$` 符号。
    *   **含义：** `$t0` 代表“编号为 t0 的寄存器”。如果不加 `$`，汇编器可能会以为 `t0` 是一个变量名或者一个常量标签，从而报错。

### 2. 深度解析：这两个例子的“方言”对比

我们可以把这两句代码翻译成“人话”，你就会发现它们意思其实是一样的：

#### 例子 1 (通用风格)
```assembly
ADD R1, R2, R3
```
*   **翻译：** 喂 CPU，请把 **寄存器2** 和 **寄存器3** 里的数字拿出来相加，结果存到 **寄存器1** 里。
*   **R** 代表 **R**egister（寄存器）。

#### 例子 2 (MIPS 风格)
```assembly
add $t1, $t0, $s1
```
*   **翻译：** 喂 CPU，请把 **寄存器t0** 和 **寄存器s1** 里的数字拿出来相加，结果存到 **寄存器t1** 里。
*   **$** 的作用：就像微博或推特里的 **@** 符号。
    *   写 `User` 可能只是个单词。
    *   写 `@User` 就代表这是一个用户。
    *   同理，写 `t0` 可能是个标签，写 `$t0` 就特指那个寄存器。

### 3. 补充知识：MIPS 里的 t 和 s 是什么？

在第二个例子（MIPS）中，寄存器名字不仅有 `$`，后面还分 `t` 和 `s`，这也是考试常考的细节：

*   **$t0 - $t9 (Temporary Registers)：** **临时寄存器**。
    *   **生活比喻：** 草稿纸。用完就扔，没人保证这上面的数据会一直保留。如果不小心被覆盖了，也就覆盖了。
*   **$s0 - $s7 (Saved Registers)：** **保存寄存器**。
    *   **生活比喻：** 档案袋。这种寄存器里的值非常重要，如果你要用这个寄存器，必须先把原来的值备份好，用完还得给人家还原回去。

### 总结

*   **$ 是什么：** MIPS 汇编语言中**寄存器的标识符**。
*   **为什么第一个没有：** 因为那是**通用教学代码**，直接用 `R` 开头，为了看着方便。
*   **为什么第二个有：** 因为那是**严格的 MIPS 代码**，语法强制要求必须加 `$`。
